# 🗺️ 小说视觉化阅读助手

以章节为时间轴的动态小说沙盒 — 上传 TXT 小说 → AI 自动提取人物/关系/地点 → 实时可视化。

---

## 📖 使用教程

### 环境准备

1. **安装 Python** 3.10+
2. **克隆/下载项目** 后，在项目根目录创建虚拟环境并安装依赖：

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate

pip install -r requirements.txt
```

3. **配置 LLM API Key**（如不配置，仍可正常阅读，仅 AI 解析功能不可用）：

```bash
# Windows PowerShell
$env:LLM_API_KEY="sk-你的key"

# Mac / Linux
export LLM_API_KEY="sk-你的key"
```

默认使用 DeepSeek API，如需切换可额外配置：

```bash
$env:LLM_BASE_URL="https://api.openai.com/v1"   # 或你的兼容地址
$env:LLM_MODEL="gpt-4o-mini"                     # 模型名
```

### 启动服务

需要**同时启动两个终端**：

**终端 1 — 后端服务（FastAPI）**：

```bash
venv\Scripts\python -m uvicorn backend.app.main:app --reload --port 8000
```

看到 `Application startup complete` 即启动成功，API 运行在 http://localhost:8000。

**终端 2 — 前端页面（Streamlit）**：

```bash
venv\Scripts\python -m streamlit run frontend/app.py --server.port 8501
```

自动在浏览器打开 http://localhost:8501。

### 使用流程

```
① 打开前端页面 → ② 上传 TXT 小说 → ③ 侧边栏点击「打开阅读」
   → ④ 左侧阅读正文 → ⑤ 右侧查看人物卡 / 关系网 / 时间轴
```

**详细步骤：**

1. **导入小说**
   - 在前端首页点击「选择 TXT 小说文件」
   - 可选填写小说名称（留空则自动使用文件名）
   - 点击「导入」按钮
   - 成功后自动创建 `data/novels/{小说名}_world/` 目录结构

2. **开始阅读**
   - 侧边栏「已导入的小说」列表中点击「打开阅读」
   - 进入左右分栏阅读模式
   - 左侧 70%：章节正文 + 上一章/下一章导航
   - 右侧 30%：可视化面板（三个标签页）

3. **查看解析结果**
   - **🎴 人物卡**：每翻到新章节自动触发 AI 解析，已解析的人物在此展示
   - **🕸️ 关系网**：已解锁人物的关系力导向图，节点可拖拽，悬停查看关系详情
   - **📋 时间轴**：已解析章节的事件列表

> **注意**：AI 解析需要配置 `LLM_API_KEY` 环境变量后才生效。未配置时仍可正常阅读，解析功能会自动跳过。

## 🎯 核心功能

```
                      ┌──────────────┐
                      │  TXT 小说导入 │
                      └──────┬───────┘
                             ▼
                 ┌───────────────────────┐
                 │  AI 章节流式解析引擎    │
                 └───────────┬───────────┘
                             ▼
          ┌──────────────────┼──────────────────┐
          ▼                  ▼                  ▼
   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
   │ 动态人物卡  │    │ 动态关系网  │    │ 动态大地图  │
   └──────┬──────┘    └──────┬──────┘    └──────┬──────┘
          ▼                  ▼                  ▼
   [头像/外貌锁定]    [双向关系时序变更]    [事件坐标/场景CG]
```

### 1. 📖 章节时间轴驱动引擎

章节流动控制：整个系统的"时间轴"即小说的"章节数"。随着用户点击"下一章"或滚动阅读，系统会发出当前章节的 `chapter_id`。

状态触发器：每一个章节是一个状态节点。当进入新章节时，系统在后台比对当前状态与上一章状态，动态解锁新的人物、地标，或更新已有关系。

### 2. 🎴 随时间演变的人物卡

- **人物发现**：自动提取新出场人物，生成包含姓名、身份、当前境界/阵营的初始卡片
- **外貌特征锁定**：AI 提取人物外貌特征（发色、瞳色、标志性道具），用于后续一致的图像生成
- **成长与状态演变**：人物卡信息是"活的"——境界突破、受伤黑化等状态实时更新

### 3. 🕸️ 动态人物关系网

- **实体关系抽取**：AI 自动分析本章中人与人之间的互动，提取双向关系
- **时序关系网络图**：使用 pyvis 渲染交互式力导向图，节点可拖拽，点击连线查看关系变迁历史

### 4. 🗺️ 递进式动态大地图（P1 延后）

- **地标解锁**：随主角旅程推进，大地图从迷雾一步步点亮
- **场景与事件 CG**：AI 根据环境描述生成场景图，高潮事件自动生成事件插画

### 5. 🔀 剧情变量模拟器（P2 延后）

- **蝴蝶效应模拟**：在关键节点添加变量，AI 推演分支剧情
- 属于进阶功能，基础体验跑通后开发

## ⚙️ 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 后端框架 | FastAPI + Uvicorn | RESTful API 服务 |
| 数据校验 | Pydantic v2 | 强类型模型 + LLM 结构化输出 |
| LLM 接口 | Instructor + OpenAI SDK | 类型安全的 AI 调用 |
| 前端框架 | Streamlit | 快速交互式界面 |
| 可视化 | pyvis | 关系网力导向图 |
| HTTP 客户端 | httpx | 前后端通信 |
